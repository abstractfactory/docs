Installing with pip

This is what you have

  • Windows >= 7, x64
  • link Visual C++ 2012 redist x64
  • link Python == 2.7.6, x64
  • link pip >= 1.5.0
  • link git >= 1.9.0

This is what you run

$ pip install git+https://github.com/abstractfactory/pifou_beta1.git  git+https://github.com/abstractfactory/pigui_beta1.git git+https://github.com/abstractfactory/piapp_beta1.git

This is what you do

$ python .\piapp\willywork\main.pyw

Where .\ refers to where pip installed Pipi; most commonly the site-packages of your Python installation.

c:\Python27\Libs\site-packages\piapp

Dependencies

Willywork is a debug application to test whether Pipi will work. It may complain about missing dependencies; if so, go ahead and also install the Pipi Dependency Module.

$ pip install git+git://github.com/abstractfactory/pidep.git

For convenience, dependencies have been encapsulated within this one package. If you decide later to use your own dependencies, well, then you are free to do so. Here they are.

  • link Open Metadata 0.5
  • link PyQt5 Built using Qt 5.1.1
  • link PyZMQ Built using libzmq 4.0.3
  • link Msgpack 0.4.1
